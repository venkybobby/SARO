# Compliance Guard Skill

## Trigger Conditions
Activate for changes to `routers/output_audit.py`, `routers/reports.py`, any file containing `compliance`, `certif`, `EU AI Act`, `NIST`, `ISO 42001`, or `AIGP`, and all files under `docs/`.

## SARO Compliance Posture (non-negotiable)

Reference: @docs/COMPLIANCE_CLAIMS_MATRIX.md

| What SARO Does | What SARO Never Does |
|---|---|
| Provides risk scores as evidence | Issues compliance certificates |
| Generates audit-trail documentation | Guarantees regulatory outcomes |
| References framework requirements | Acts as a certification body |
| Supports human reviewers | Replaces human sign-off |
| Logs immutable TRACE timelines | Modifies client system records |

## Forbidden Language in Code & Docs

The following strings must never appear in source code comments, API responses, UI copy, or documentation:

```
"SARO certifies"
"compliant with EU AI Act"
"ISO 42001 certified"
"NIST RMF compliant"
"audit passed"
"regulatory approval"
"certified output"
```

If you find these strings, replace them with the approved alternatives:

| Forbidden | Approved |
|---|---|
| "certifies compliance" | "provides evidence supporting compliance review" |
| "compliant with X" | "evaluated against X framework criteria" |
| "audit passed" | "audit evidence generated — human review required" |

## Audit Trail Requirements

- Every scan must produce an immutable TRACE record written to `audit_logs` table.
- Audit records must be append-only — no UPDATE or DELETE on `audit_logs`.
- TRACE records must include: `scan_id`, `client_id`, `timestamp_utc`, `risk_score`, `rule_pack_version`, `reviewer_id` (null until human signs off).
- The `reviewer_id` field must only be set by an authenticated human action, never auto-populated.

## Report Generation Rules

- Reports are read-only exports — PDF/JSON of TRACE data.
- Reports must include the disclaimer: *"This report is audit evidence generated by SARO. It does not constitute regulatory certification."*
- Reports must not include definitive pass/fail verdicts — only scored risk ranges with recommended actions.

## Framework References

When code references a framework, use only these approved patterns:

```python
# NIST AI RMF 1.0
framework_ref = "NIST-AI-RMF-1.0"  # not "NIST-certified"

# EU AI Act
framework_ref = "EU-AI-Act-evidence"  # not "EU-AI-Act-compliant"

# ISO 42001
framework_ref = "ISO-42001-lifecycle-ref"  # not "ISO-42001-certified"
```
