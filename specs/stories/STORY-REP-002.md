# STORY-REP-002: Implement export menu (PDF, CSV, Share Link)

**Status:** ready
**Screen/Area:** Reports / Analytics

## Goal
Wire the export buttons so users can download or share the report. PDF export includes charts, CSV export contains raw data, and Share Link generates a read-only public link.

## Acceptance Criteria (Given/When/Then — required before /story will run)
- AC-1: Given the user clicks "Export as PDF", When the export completes, Then a PDF file downloads with all visible charts, report title, and date range in the filename
- AC-2: Given the user clicks "Export as CSV", When the export completes, Then a CSV file downloads containing the underlying data for all five charts (separate sections or sheets)
- AC-3: Given the user clicks "Share Link", When the action completes, Then a read-only shareable URL is copied to clipboard and a toast confirms the action
- AC-4: Given a chart contains no data, When export is attempted, Then the export still completes but includes a note "No data for this period"
- AC-5: Given the user exports a large report (>100 rows), When the export completes, Then the file size is <10MB

## Edge Cases
- Export is triggered while chart data is still loading → disable buttons until data is ready, or show a "Generating..." state
- User's browser blocks downloads → show an error message explaining clipboard/permission requirements
- Generated share link is accessed by a non-authenticated user → render a read-only anonymous view or prompt login

## Out of Scope
- Email-based report distribution (covered in a future Notifications epic)
- Custom branding in exported PDFs (covered in Organization Settings)
- Expiring share links (can be added in v8.1)

## Non-Functional Requirements
- PDF generation latency: <5s for a full report
- Share link generation: <1s (using short-URL service or UUID)
- CSV file size: <5MB for typical reports
- All exports include a "Generated by SARO" footer and timestamp for auditability
